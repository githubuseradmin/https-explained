# https-explained

Интерактивный анимированный одностраничный разбор того, **что на самом деле
происходит, когда вы вводите адрес `https://` и нажимаете Enter** — от разбора URL
и до момента, когда браузер рисует пиксели на экране.

Он сделан для тех, кто хочет получить *правильную* мысленную модель жизненного цикла
запроса: DNS, TCP, TLS, HTTP и рендеринг — каждый этап как отдельный шаг с
анимированной диаграммой движущихся пакетов, точным объяснением, указанием
соответствующего уровня OSI / порта / протокола и заметкой «что может пойти не так».

Чистый HTML / CSS / JavaScript. **Без сборки, без фреймворков, без рантайм-зависимостей.**
Единственный внешний ресурс — шрифт Google Fonts, подключённый через `<link>`.

---

## Чему он учит

Приложение проходит семь этапов HTTPS-запроса:

1. **Разбор URL** — разделение `https://example.com:443/docs` на схему / хост / порт / путь.
2. **Разрешение DNS** — рекурсивный резолвер → корневой → TLD → авторитетный сервер, возвращающий IP (запись A/AAAA), по порту **53**.
3. **Трёхэтапное рукопожатие TCP** — `SYN` / `SYN-ACK` / `ACK` для установления надёжного соединения по порту **443**.
4. **Рукопожатие TLS** — `ClientHello`, `ServerHello` + сертификат, обмен ключами ECDHE, `Finished` → сеансовые ключи. Отмечено, что **TLS 1.3 завершается за 1-RTT**, тогда как **TLS 1.2 требовал 2-RTT**, а эфемерный ECDHE даёт **прямую секретность** (forward secrecy).
5. **Зашифрованный HTTP-запрос** — метод + заголовки, отправленные внутри туннеля TLS; сравниваются **HTTP/1.1** (по одному запросу за раз), **HTTP/2** (мультиплексирование) и **HTTP/3** (поверх QUIC/UDP).
6. **HTTP-ответ** — строка статуса + заголовки + тело (HTML).
7. **Рендеринг в браузере** — HTML → DOM, CSS → CSSOM, дерево рендеринга → раскладка (layout) → отрисовка (paint).

На каждом шаге также раскрывается **аспект безопасности** — фишинг через
омоглифы/Punycode, подмена DNS и DoH/DoT, DoS-атака SYN-flood, MITM и проверка
сертификата, флаги защищённых cookie, HSTS/CSP и политика одного источника
(same-origin) — так что путешествие заодно становится экскурсией по тому, где в стеке
живёт безопасность.

## Возможности

- **Анимированная SVG-диаграмма** — пакеты перемещаются между клиентом и
  резолвером/сервером (а для DNS — через узел корень/TLD/авторитетный сервер) с
  помощью Web Animations API.
- **Кнопки Назад / Вперёд / Воспроизведение / Повтор** плюс кликабельная полоса
  прогресса с шагами и заполняющийся индикатор.
- **Таймлайн «с высоты птичьего полёта»** — компактный обзор всех семи этапов,
  который можно охватить одним взглядом и кликом перейти к любому шагу.
- **Глоссарий** — определения простым языком для каждой аббревиатуры (DNS, TCP, TLS,
  RTT, ECDHE, MITM, HSTS, CSP, QUIC, …), чтобы новичок никогда не оставался в догадках.
- **Прямые ссылки на шаги** — текущий шаг хранится в хэше URL (например,
  `#step/tls`), поэтому можно сослаться сразу на нужный этап, а кнопки «Назад»/«Вперёд»
  браузера переключают шаги.
- **Поддержка клавиатуры** — `←` / `→` (или `h`/`l`, `j`/`k`) для навигации, `Space`
  для воспроизведения/паузы, `R` для повтора, `Home`/`End` к первому/последнему шагу.
  На экране показана подсказка с горячими клавишами; есть ссылка пропуска (skip link)
  и видимые рамки фокуса.
- **Удобство на мобильных** — одна колонка на маленьких экранах, две колонки на
  широких; полоса шагов прокручивается по горизонтали и держит активный шаг в поле
  зрения; нет горизонтального переполнения.
- **Учитывает `prefers-reduced-motion`** — если ОС просит уменьшить движение, пакеты
  мгновенно перескакивают к месту назначения вместо анимации, а мигающий курсор
  останавливается.
- **Доступность** — семантические ориентиры (landmarks), атрибуты `aria-label` и
  состояния `aria-current`/`aria-pressed` на элементах управления, подпись с
  `aria-live`, озвучивающая каждый пакет, и осмысленное описание `role="img"` для
  диаграммы.

## Запуск локально

Это статический сайт, поэтому достаточно открыть файл:

```bash
# Вариант A: открыть напрямую
#   двойной клик по index.html, либо
open index.html            # macOS
start index.html           # Windows
xdg-open index.html        # Linux

# Вариант B: поднять сервер (приятнее; избегает особенностей file://)
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

Никакой установки, никакого `npm install`, ничего компилировать не нужно.

## Тесты

Содержимое и чистая логика лежат в `steps.js` **без каких-либо зависимостей от DOM**,
поэтому их можно покрыть юнит-тестами под Node без браузера и без сторонних библиотек.
Набор тестов использует встроенный тест-раннер Node:

```bash
node --test
```

Он проверяет, что содержимое остаётся внутренне согласованным — семь шагов в нужном
порядке, у каждого пакета корректное направление/узел, вспомогательный узел использует
только шаг DNS, глоссарий покрывает каждую аббревиатуру, на которую опираются шаги,
вспомогательные функции работы с хэшем URL обратимы для каждого шага, и что **в
отрисовываемое содержимое не попадают сырые кириллические символы** (пример с
омоглифом выводится как HTML-сущность, поэтому исходник остаётся чистым ASCII).

Можно также проверить синтаксис скриптов, ничего не запуская:

```bash
node --check steps.js
node --check app.js
node --check steps.test.js
```

Пример вывода:

```
$ node --test
✔ there are exactly seven steps in canonical order
✔ every step has the required fields and non-empty content
✔ hashForIndex and indexFromHash round-trip for every step
✔ the glossary covers the core acronyms the steps lean on
✔ no raw Cyrillic characters leak into rendered step content
ℹ tests 15
ℹ pass 15
ℹ fail 0
```

## Использование в обучении / в курсе

Этот материал задуман как **учебное пособие для вводного модуля по сетям или
веб-безопасности**. Он намеренно дружелюбен к новичкам и самодостаточен:

- **Показывайте по одному шагу за раз.** Каждый этап — самостоятельный «слайд»:
  откройте страницу, нажимайте `→` и комментируйте диаграмму, пока движутся пакеты.
- **Давайте прямые ссылки из конспекта или LMS.** У каждого шага стабильный хэш URL
  (`#step/dns`, `#step/tls`, …). Вставляйте эти ссылки в слайды, вики или задания,
  чтобы студенты попадали ровно туда, куда нужно.
- **Используйте глоссарий как раздаточный словарь.** Он сопоставляет каждой
  аббревиатуре одно простое предложение — готовая шпаргалка для опроса или повторения.
- **Используйте заметки о безопасности как темы для обсуждения.** Каждая карточка
  «что может пойти не так» — естественный повод для вопроса («почему обычный DNS
  поддаётся подмене и что это исправляет?»).
- **Расширяйте как упражнение.** Поскольку каждый шаг — это одна запись в массиве
  `STEPS` в `steps.js`, задание добавить шаг (например, HTTP/3 поверх QUIC) или
  исправить упрощение становится компактной и приятной задачей по программированию,
  а набор тестов сразу даёт обратную связь.

Раздел «Заметки о точности» ниже стоит читать вместе с приложением, чтобы упрощения
были явными, а не вводили в заблуждение.

## Развёртывание на GitHub Pages

Поскольку `index.html` лежит в корне репозитория, развёртывание идёт как есть:

1. Запушьте репозиторий на GitHub.
2. Перейдите в **Settings → Pages**.
3. В **Build and deployment** выберите **Source = Deploy from a branch**.
4. Выберите ветку (например, `main`) и папку **`/ (root)`**, затем нажмите **Save**.
5. Сайт станет доступен по адресу `https://<user>.github.io/<repo>/` в течение минуты.

Никакого GitHub Action или конфигурации сборки не требуется.

## Заметки о точности (намеренные упрощения)

Это обучающий инструмент. Модель верна по сути, но в ряде мест намеренно упрощена:

- **Рукопожатие TLS 1.3.** Реальный поток таков: `ClientHello` → `ServerHello`,
  `EncryptedExtensions`, `Certificate`, `CertificateVerify`, `Finished` (сервер),
  затем `Finished` клиента. Приложение сворачивает это до четырёх иллюстративных
  сообщений и подчёркивает главное: сертификат подтверждает личность сервера, ключи
  выводятся через **ECDHE** (поэтому общий секрет никогда не передаётся, и каждый
  сеанс получает **прямую секретность**), а рукопожатие занимает **1-RTT**. TLS 1.2
  требует лишнего кругового обхода (2-RTT); 1.3 также поддерживает возобновление
  **0-RTT** (с оговорками о повторах), что упоминается, но не анимируется.
- **DNS.** Реальное разрешение включает кэширование на нескольких уровнях, возможные
  цепочки CNAME и отдельные запросы к корневым, TLD- и авторитетным серверам.
  Приложение показывает один типовой круговой обход плюс обход иерархии. Показанный IP
  (`93.184.216.34`) — известный адрес, исторически связанный с `example.com`, и
  используется исключительно как иллюстративная константа; никаких реальных обращений
  не происходит.
- **Порты.** DNS показан на **53** (UDP с откатом к TCP для больших ответов); HTTPS —
  на **443**. Это значения по умолчанию от IANA; `:443` в URL обычно опускают, так как
  это порт по умолчанию для схемы `https`.
- **Уровни TCP / TLS.** TLS часто описывают как находящийся между транспортным и
  прикладным уровнями; панель помечает его «Presentation / Session (above TCP)» как
  прагматичное, а не строгое сопоставление с OSI.
- **Версии HTTP.** Приложение ссылается на HTTP/2 поверх TLS и отмечает различия между
  HTTP/1.1, HTTP/2 и HTTP/3. HTTP/3 работает поверх **QUIC (UDP)**, а не TCP, и
  объединяет транспортное и TLS-рукопожатие, что изменило бы более ранние шаги;
  анимация этого варианта выходит за рамки разбора.
- **Пример с омоглифом.** Фишинговая карточка показывает кириллическую `а` (U+0430)
  рядом с латинской `a` (U+0061). В исходнике она записана как HTML-сущность `&#1072;`,
  чтобы файлы оставались чистым ASCII, но браузер при этом всё равно рисовал
  «двойника».

Если вы заметите неточность — это стоит исправить: корректность здесь и есть цель.

## Технологии

- Чистый JavaScript (ES2015+), без зависимостей.
- Небольшая обёртка экспорта в стиле UMD позволяет `steps.js` работать и как
  глобальная переменная браузера, и как модуль CommonJS, поэтому один и тот же файл
  питает и страницу, и тесты Node — по-прежнему без шага сборки.
- SVG для диаграммы; Web Animations API для движения пакетов.
- Пользовательские свойства CSS для тем; CSS Grid для раскладки.
- Google Fonts: [Inter](https://fonts.google.com/specimen/Inter) (интерфейс) и
  [JetBrains Mono](https://fonts.google.com/specimen/JetBrains+Mono) (терминальный вид).
